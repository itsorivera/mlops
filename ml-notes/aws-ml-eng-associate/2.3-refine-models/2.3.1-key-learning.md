## Key Terms and Concepts to Master

### 1. Model Diagnostics & Performance

- **Model bias**: The error identified when a model can't learn the true relationship between input features and target variables. A higher model bias indicates that a model might be too simple. Common causes: model is too simple, incorrect modeling/feature engineering, inherited bias from the dataset. High bias can lead to underfitting.
  - _Note on "Bias": Mathematical vs. Societal. In model evaluation, Bias refers to Statistical Bias (the algorithm's inability to capture real data complexity, leading to underfitting). For example: If the true relationship between years of experience and salary is an exponential curve, but you use Linear Regression (which only draws straight lines), the model will never fit the curve well. That gap is statistical bias. This is completely different from Societal/Data Bias, which occurs when training data reflects human prejudices or historical inequalities (which AWS addresses with tools like Amazon SageMaker Clarify)._
- **Model variance**: The sensitivity of the model to different training data (susceptibility to noise). A high variance model might be too complex for the training data or might have memorized every data point. Common causes: model is too complex, too much irrelevant data (noise) in the dataset, training for too long. High variance can lead to overfitting.
  - _Example:_ Imagine predicting house prices based on size. The data has natural "noise" (some small houses might be randomly more expensive due to unique factors). If you use a highly complex algorithm (like a 15th-degree polynomial or an overly deep Decision Tree), it will draw a wildly squiggly line that connects and passes _exactly_ through every single training point, memorizing even the anomalies. When you try to predict the price of a new, unseen house, this squiggly line will give a wildly inaccurate prediction. The model didn't learn the general upward trend; it just memorized the specific noise of that single training dataset.
- **Bias-Variance Trade-off**: The goal during evaluation is model balance. Improving one metric often negatively impacts the other. The objective is _not_ to find the absolute lowest possible bias and variance simultaneously, but to find the optimal balance.
  - 🆚 **Quick Comparison to Avoid Confusion:**
    - **Bias** = _Rigid/Too Simple_. The model is "lazy" and misses the pattern. It performs terribly on BOTH training data and new data (**Underfitting**).
    - **Variance** = _Overly Sensitive/Too Complex_. The model is a "memorizer". It performs perfectly on training data, but terribly on new data because it memorized the noise (**Overfitting**).
  - 💡 **How to detect them (The Golden Rule):**
    - **Bias is evidenced during TRAINING:** If your model has high error on the data it is actively learning from, it has high bias (it's too simple to learn).
    - **Variance is evidenced during VALIDATION:** You cannot detect variance just by looking at training data (where the error will be artificially low). Variance only reveals itself when the model processes _new, unseen data_ and the error spikes significantly compared to the training error.
- **Overfitting**: A modeling error that occurs when a function is too closely fit to a limited set of data points, performing poorly on unseen data.
- **Underfitting**: A modeling error that occurs when a function cannot adequately capture the underlying structure of the data.
- **Model evaluation metrics**: Formulas and techniques used to measure the performance of a model.

### 2. Mitigation & Optimization Techniques

- **Regularization techniques**: Methods used to reduce the complexity of a model to prevent overfitting (e.g., L1/Lasso, L2/Ridge, Dropout).
- **Feature selection**: The process of selecting a subset of relevant features (variables, predictors) for use in model construction.
- **Iterative model pruning**: A technique to reduce model size and inference cost by systematically removing weights, neurons, or layers.
- **Catastrophic forgetting**: The tendency of an artificial neural network to completely and abruptly forget previously learned information upon learning new information.

### 3. Ensemble Methods

- **Ensemble methods**: Machine learning technique that combines several base models in order to produce one optimal predictive model.
- **Boosting**: A sequential technique which adjusts the weight of an observation based on the last classification (e.g., XGBoost, AdaBoost).
- **Bagging**: (Bootstrap Aggregating) trains models in parallel on random subsets of the data with replacement (e.g., Random Forest).
- **Stacking**: Combining multiple classification or regression models via a meta-classifier or a meta-regressor.

### 4. Hyperparameter Tuning

- **Hyperparameters**: Parameters whose values are used to control the learning process, set before training begins.
- **Hyperparameter tuning techniques**: Strategies for finding the best set of hyperparameters (e.g., Grid Search, Random Search, Bayesian Optimization).
- **Automated hyperparameter optimization (HPO)**: Using automated algorithms to search for optimal hyperparameter configurations.

### 5. Transfer Learning & Foundation Models

- **Fine-tuning**: Taking a pre-trained model and training it further on a smaller, task-specific dataset.
- **Pre-trained models**: Models created by someone else to solve a similar problem, saving time and computational resources.

### 6. AWS Specific Tools & Services

- **Amazon SageMaker JumpStart**: A machine learning hub with foundation models, built-in algorithms, and pre-built ML solutions.
- **Amazon Bedrock**: A fully managed service that makes foundation models accessible via API.
- **Amazon SageMaker Model Registry**: A central repository to manage model versions, approval statuses, and metadata for tracking repeatability and audits.

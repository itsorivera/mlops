# Preventing Overfitting and Underfitting

## Remediating Overfitting

The following techniques are commonly used to prevent overfitting and improve a model's ability to generalize to unseen data:

### 1. Early Stopping

!["Early Stopping"](/aws-ml-associate/assets/2.3.2_remediatingOverfit_e.png)

Early stopping halts the training process before the model begins to learn the noise in the data. By monitoring the validation dataset's performance alongside the training dataset, training is paused when validation performance stops improving (plateaus) or starts worsening for a certain number of epochs. This identifies the optimal stopping point before the model overfits.

### 2. Pruning

Pruning removes unnecessary features or weights in neural networks that do not significantly contribute to the training process. By identifying redundant features and reducing the emphasis on irrelevant connections, the model avoids learning from noise.

### 3. Regularization

Regularization prevents models from overfitting by penalizing high weight values:

- **Dropout:** Frequently used in neural networks, it randomly disables a subset of neurons during training epochs. This prevents the network from relying too heavily on specific nodes, promoting more robust feature learning.

!["Dropout"](/aws-ml-associate/assets/2.3.2_remediatingOverfit_r.png)

- **L1 Regularization (Lasso):** Drives the weights of less important features exactly to zero, effectively acting as feature selection. It produces sparse models that are less prone to noise.

!["L1"](/aws-ml-associate/assets/2.3.2_remediatingOverfit_r%20(1).png)

- **L2 Regularization (Ridge / Weight Decay):** Penalizes large weights, resulting in smaller, more evenly distributed weight values across the model. This stabilizes the weights, particularly when input features are highly correlated, producing a smoother and generalized model.

### 4. Data Augmentation

!["Early Stopping"](/aws-ml-associate/assets/2.3.2_remediatingOverfit_d.png)

Data augmentation artificially expands the training dataset by applying transformations to the existing data (e.g., rotating, scaling, flipping, cropping, or adding noise to images). This increases data diversity, exposing the model to more real-world variations and reducing the likelihood of overfitting.

### 5. Model Architecture Simplification

Simplifying the model architecture (e.g., reducing the number of layers, neurons, or parameters) directly reduces its capacity to memorize the training data, helping to correct and prevent overfitting.

---

**Key Takeaway:** You can prevent a model from overfitting (memorizing training data) by simplifying it (Model Simplification, Pruning), penalizing complexity (L1/L2 Regularization, Dropout), knowing when to stop training (Early Stopping), or exposing it to a wider variety of scenarios (Data Augmentation).

## Remediating Underfitting

The following techniques are designed to correct underfitting and ensure the model is flexible enough to learn the underlying relationships in the data:

### 1. Train for an Appropriate Length of Time

!["Apropiate long time"](/aws-ml-associate/assets/2.3.2_remediatingUnderfit_.png)

Complex models (like deep neural networks) require enough time (epochs) to converge on optimal weight values. If training is stopped prematurely, the model will underfit. You must continuously monitor performance metrics on a validation set and allow training to continue until performance plateaus.

**Note:** Calibrating training duration correctly is essential to solving _both_ underfitting (too short) and overfitting (too long).

### 2. Increase Data Points

!["Increase Data Points"](/aws-ml-associate/assets/2.3.2_remediatingUnderfit_%20(1).png)

If the training data is too small or unrepresentative, the model lacks information to establish correct weights. Increasing the dataset size—whether by gathering more real-world data or using data augmentation to create synthetic examples—gives the model sufficient patterns to learn from.

### 3. Increase Model Flexibility (Complexity)

You can give a model more capacity to capture intricate patterns through several methods:

- **Add New Domain-Specific Features:** Combine or introduce variables that give the model more context (e.g., calculating a new `volume` feature from `length`, `width`, `height`).
- **Add Cartesian Products (Feature Crosses):** Combine categories from existing variables to capture interactions (e.g., combining `State` and `Population Density` to create `urban_Washington`).
- **Change Feature Engineering:** Adjust processing techniques to capture more complex inputs (e.g., increasing `n-gram` sizes in NLP).
- **Decrease Regularization:** Excessive regularization stifles a model's ability to adjust weights. Reducing regularization strength allows parameter weights the freedom needed to map data complexity accurately.

---

**Key Takeaway:** You can fix an underfitting model (one that is too simplistic) by increasing its complexity and flexibility (adding features, reducing regularization), ensuring it has enough data to learn from, and training it for an adequate amount of time.
